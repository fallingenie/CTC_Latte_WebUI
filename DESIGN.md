# CTC Latte WebUI 디자인 시스템

## 제품 방향

CTC Latte WebUI는 학생, 교사, 일반 사용자가 전 세계 CMIP6 미래 기후자료를 지도에서 탐색하고 검증 가능한 형태로 저장하는 교육·연구용 웹 앱이다.

첫인상에서 남겨야 할 한 문장은 다음과 같다.

> 누구나 쉽게 사용할 수 있지만, 결과는 연구자가 다시 확인할 수 있을 만큼 정직한 기후 지도.

## 디자인 원칙

1. **지도가 먼저다.** 지도, 위치, 조회 조건, 결과가 한 작업 흐름 안에서 이어져야 한다.
2. **과학적 상태를 숨기지 않는다.** 조회 중, 결측, 원자료 격자, 보정값, 보정 전 값은 색과 문장으로 분명히 구분한다.
3. **일반 사용자의 언어를 쓴다.** 전문 식별자는 필요한 곳에서만 보조 표기로 사용하고 행동 문구는 한국어로 쓴다.
4. **장식보다 계층을 우선한다.** 카드와 그림자를 남발하지 않고, 여백·선·타이포그래피로 구조를 만든다.
5. **모든 화면 크기에서 같은 작업을 끝낼 수 있어야 한다.** 320px 모바일부터 와이드 데스크톱까지 가로 스크롤 없이 조회와 저장이 가능해야 한다.
6. **색만으로 상태를 전달하지 않는다.** 아이콘, 제목, 보조 문구를 함께 사용한다.
7. **실제 자료가 아니면 수치로 표시하지 않는다.** 화면·비교·그래프·내보내기의 모든 값은 `ctwebui`, `ctcapsule` 또는 raw CMIP6 응답에서만 생성한다. 응답이 없으면 결측 또는 연결 상태를 표시한다.

## 미학

- 방향: 정제된 과학 도구, Apple Maps와 현대적인 현장 연구 노트의 중간
- 장식 수준: 최소한의 깊이만 사용하는 의도적 절제
- 레이아웃: 지도 중심의 엄격한 그리드, 작은 화면에서는 작업 순서대로 재배치
- 모션: 이해를 돕는 120~220ms 상태 전환만 사용
- 모서리: 반복 항목과 패널은 최대 8px, 원형은 지도 핀과 순수 아이콘 버튼에만 사용

## 색상

| 역할 | 값 | 용도 |
|---|---:|---|
| 바탕 | `#F4F6F5` | 앱 배경 |
| 표면 | `#FFFFFF` | 입력·패널·대화상자 |
| 잉크 | `#16201D` | 주요 텍스트 |
| 보조 잉크 | `#5F6D68` | 설명·단위 |
| 경계 | `#D8DEDB` | 구조 구분선 |
| 핵심 녹색 | `#176B55` | 주요 행동·선택·정상 상태 |
| 깊은 녹색 | `#0E3F34` | 브랜드·주요 버튼 |
| 지도 파랑 | `#19779A` | 강수·지도 상호작용 |
| 기온 주황 | `#D65A3A` | 고온·열지수 |
| 주의 황색 | `#B77708` | 부분 자료·주의 |
| 보라 회색 | `#6D6483` | 보정 전 원자료 비교 |

보라색 그라디언트와 단일 색조 배경은 사용하지 않는다. 의미가 있는 지표와 상태에만 색을 쓴다.

## 타이포그래피

- 기본: `Pretendard Variable`, `Pretendard`, `SUIT Variable`, `Noto Sans KR`, `Segoe UI`
- 숫자: 같은 글꼴의 tabular 숫자 기능 사용
- 제목: 700~800, 본문: 400~500, 조작 요소: 650~700
- 페이지 제목: `clamp(1.65rem, 2.6vw, 2.4rem)`
- 패널 제목: 1.05~1.2rem
- 본문: 0.9~1rem
- 작은 보조 문구: 최소 0.75rem
- 글자 간격: 0

## 간격과 크기

- 기본 단위: 4px
- 간격 단계: 4, 8, 12, 16, 20, 24, 32, 40
- 데스크톱 최대 콘텐츠 폭: 1600px
- 터치 조작 영역: 최소 44x44px
- 입력 높이: 44px
- 지도 높이: 화면 높이에 반응하되 데스크톱 340~500px, 모바일 260~360px

## 화면 구조

화면별 기본 디자인 방향은 학생 탐색 A(Field Atlas), 교사용 수업 B(Climate Classroom), 일반 요약 C(Living Globe)로 구분한다. 세 화면은 구조가 달라도 같은 데이터·아이콘·접근성 계약을 공유한다.

### 학생 탐색

- 왼쪽: 예시와 조회 조건
- 오른쪽 위: 세계 지도
- 오른쪽 아래: 조회 상태와 5개 통합 기후지표
- 실제 자료 두 조건을 저장하고 지표별 차이를 비교
- 탐구 주제와 발견 내용을 활동 기록으로 저장
- 960px 이하: 지도 → 조건 → 결과 순서

### 교사용 수업

- 수업 상태는 짧은 요약 띠로 표시
- 수업명, 학습 목표, 장소, 날짜, 시나리오, 모델을 한 흐름에서 구성
- 실제 자료 비교 지점을 최대 3곳까지 수업 활동지에 포함
- 현재 조건을 학생용 링크로 전달하고 CSV, PDF, PNG로 내보내기
- 사용자에게 내부 실행·검증 용어는 노출하지 않음

### 일반 요약

- 날짜, 시기, 시나리오, CMIP6 모델, 위치를 간단한 조작만으로 변경
- 날짜는 연도·월·일을 모두 선택하고 `확인`을 눌러야 실제 조회 조건으로 반영
- 전체 앙상블과 메타데이터가 제공하는 개별 CMIP6 모델을 같은 선택 상자에서 제공
- 현재 위치 또는 세계 지도에서 선택한 좌표로 실제 자료를 다시 조회
- 핵심 지표와 쉬운 문장형 설명을 함께 표시
- 결과 항목은 중첩 카드 대신 구분선 기반 목록으로 표시
- 자세한 조건은 학생 탐색으로 자연스럽게 연결

### 내보내기

- 기간, 형식, 지표 선택은 왼쪽 제어 영역
- 그래프와 자료 범위 확인은 오른쪽 미리보기
- 모바일에서는 전체 화면 시트로 전환
- CSV, PDF, PNG는 같은 흐름에서 선택
- 직접 파일 쓰기가 확인된 경우에만 저장 완료를 표시하고, 브라우저 다운로드 방식은 요청 상태로 구분

## 컴포넌트 규칙

- 주요 버튼: 깊은 녹색 배경, 흰색 텍스트, 명령을 나타내는 Lucide 아이콘
- 보조 버튼: 흰색 또는 투명 표면, 1px 경계선
- 지도 버튼: `+`, `-`, 현재 위치 아이콘을 수직 도구막대로 배치
- 지표 아이콘: 최고기온, 최저기온, 강수량, 풍속, 열지수, 체감기온을 서로 다른 상징으로 구분
- 상태 문구: 짧은 제목과 필요한 행동을 함께 표시
- 로딩: 차단형 오버레이를 사용하되 화면을 과도하게 흐리지 않고, 경과 시간과 처리 단계를 보여준다.

## 반응형 기준

- 1280px 이상: 넓은 지도와 3열 지표
- 961~1279px: 좁은 조건 패널과 2~3열 지표
- 601~960px: 단일 열, 지도 우선
- 320~600px: 1열 입력·지표, 상단 탐색은 가로 스크롤 가능한 세그먼트

## 접근성

- 키보드 포커스는 3px 외곽선으로 표시
- `prefers-reduced-motion`을 존중
- 선택 상태는 색뿐 아니라 아이콘·경계·문구로 표시
- 지도는 방향키와 Enter/Space 조작을 지원
- 대화상자는 포커스 순환과 Escape 닫기를 지원
- 테마는 시스템 설정, 밝게, 어둡게 중 선택하며 사용자 선택을 저장한다.
- 시스템 설정 모드에서는 운영체제 색상 모드 변경을 즉시 반영한다.
