# Android 이식 기준

## 결정

첫 Android 배포 단위는 현재 공개 WebUI를 설치하는 PWA로 한다. 자료 조회, 지도, 역할별 화면, 자료판 갱신과 내보내기는 WebUI의 동일한 공개 계약을 사용한다. Android에서 기후 수치를 다시 계산하거나 별도 자료 캐시를 만들지 않는다.

Play 스토어 배포가 필요해지면 PWA를 Trusted Web Activity(TWA)로 감싼다. TWA는 전용 도메인, HTTPS, 앱 서명키와 Digital Asset Links가 준비된 뒤에만 출시한다. 이 조건이 없으면 주소 표시줄이 나타나는 일반 Custom Tab으로 열릴 수 있으므로 출시판으로 간주하지 않는다.

Capacitor는 Android WebView에서 파일 저장이나 시스템 공유가 실제 기기 회귀를 통과하지 못할 때만 사용한다. 현재 React 화면을 Kotlin 또는 Compose로 다시 작성하지 않는다.

## 보안 경계

- 접속 암호 화면은 시험 참여자의 우발적 접근을 줄이는 장치이며 사용자 인증이나 접근 권한 제어가 아니다.
- 앱과 manifest, 서비스 워커, 정적 파일에는 GCS, Google Drive, 로컬 경로, 서명키와 API 비밀값을 넣지 않는다.
- 공개 API의 호출 제한, 남용 방지와 운영 비용 보호는 서버에서 집행한다.
- 서비스 워커는 `/api/climate/` 응답을 저장하지 않는다. 오프라인 화면은 앱 셸만 제공하며 실제 기후값처럼 보이는 대체값을 만들지 않는다.
- 자료판이 바뀌면 WebUI와 동일한 계약으로 최신 자료를 다시 조회한다.

## 단계별 출하 기준

### 1단계: 설치형 PWA

- 192px, 512px, 마스크형 512px 아이콘
- 일반 요약 시작 화면과 학생·교사·일반 바로가기
- 밝은 화면, 어두운 화면, 시스템 설정 연동
- Android Chrome에서 지도 확대·축소·이동·좌표 선택
- 장시간 원자료 조회의 진행 표시, 취소, 한 번 재시도
- CSV, PDF, PNG, HTML, 학생 기록 DOCX, 교사 활동지 DOCX 저장
- 뒤로가기, 새로 열기, 화면 회전, 글자 크기 200%, TalkBack 확인
- 오프라인에서 실제 자료값을 표시하지 않는 실패 폐쇄 동작

### 2단계: TWA 시험판

- 전용 도메인과 `/.well-known/assetlinks.json`
- Android 16(API 36) 이상을 대상으로 한 AAB
- Play App Signing과 별도 업로드 키
- 패키지 이름과 앱 링크 소유권 검증
- 주소 표시줄이 없는 전체 화면 확인
- 내부·비공개 저장소 주소가 APK와 로그에 없는지 검사
- 개인정보 처리방침과 Play Data Safety 응답 일치
- 내부 시험, 비공개 시험, 공개 시험 순서의 단계별 승격

### 3단계: 선택적 네이티브 연결

- Web Share API와 Android 공유 창의 파일 형식별 회귀
- 다운로드가 실패하는 형식만 Capacitor Filesystem/Share 플러그인으로 보완
- 네이티브 연결을 사용해도 WebUI와 Backend 자료 계약은 하나로 유지

## 방식별 판단

| 방식 | 적용 판단 | 장점 | 비용과 위험 |
| --- | --- | --- | --- |
| PWA | 첫 Android 배포 | 현재 React 화면과 자료 계약을 그대로 사용하고 웹 갱신이 즉시 반영됨 | Android 실제 기기의 저장·공유·오프라인 회귀가 필요함 |
| TWA | Play 배포가 필요할 때 | 검증된 PWA를 주소 표시줄 없는 앱으로 제공함 | 전용 도메인, 앱 링크 소유권, AAB와 서명 운영이 추가됨 |
| Capacitor | 저장·공유가 실제 기기에서 실패할 때만 | 필요한 파일 기능만 네이티브로 보완할 수 있음 | WebView, 플러그인, 권한과 별도 출시 회귀가 추가됨 |
| 네이티브 재작성 | 현재 적용하지 않음 | Android 기능을 가장 세밀하게 제어할 수 있음 | 화면, 자료 계약, 내보내기와 인용 로직을 중복 구현해 불일치 위험이 큼 |

GitHub Pages는 응답 보안 헤더를 세밀하게 통제할 수 없다. TWA는 콘텐츠 보안 정책, 프레임 차단, 권한 정책과 참조 정보 정책을 직접 설정할 수 있는 전용 도메인과 호스팅에서 출시한다.

## 실제 기기 회귀

1. Chrome에서 설치한 뒤 첫 실행, 오프라인 재실행, 다시 연결, 새 판 갱신을 확인한다.
2. CSV, PDF, PNG, HTML, 학생 기록 DOCX, 교사 활동지 DOCX의 저장·열기·취소·파일명·형식을 확인한다.
3. Android 뒤로가기, 화면 회전, 앱 전환과 복귀, 장시간 조회 취소와 한 번 재시도를 확인한다.
4. TalkBack, 키보드 또는 스위치 접근, 글자 크기 200%, 밝게·어둡게·시스템 설정을 확인한다.
5. 시스템 공유가 필수 요구가 되면 Web Share API를 먼저 검증하고, 실패 형식만 네이티브 연결로 보완한다.

## 현재 환경 제약

로컬 Android SDK는 있으나 실행 가능한 가상 기기와 출시 서명키는 준비되지 않았다. 따라서 실제 Android 기기 또는 검증용 가상 기기에서 1단계 회귀를 끝내기 전에는 APK나 Play 출시 준비가 완료되었다고 표시하지 않는다.
